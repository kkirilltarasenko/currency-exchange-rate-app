# Требования к системе

**Версия документа:** 1.0  
**Дата создания:** 21.02.2026  

---

## 1. Описание проекта

### 1.1 Общее описание

Разрабатывается приложение для предоставления пользователям актуальной информации о курсах валют на основе данных,
получаемых из доступных интернет-сервисов. Приложение позволяет просматривать текущие курсы валют, анализировать
изменения курса и получать дополнительную информацию о валютных показателях.

### 1.2 Назначение системы

Система предназначена для повышения удобства получения информации о валютных курсах и анализа динамики изменения валют.

## 4. Валютная политика системы

### 4.1 Обоснование выбора валют

**Рекомендуемое решение:** Пользователь самостоятельно выбирает интересующие валюты

**Обоснование:**

- ISO стандарт содержит сотни валют
- Различные пользователи имеют разные потребности
- Приложение становится универсальным и масштабируемым
- Повышается пользовательский опыт

### 4.2 Минимальный набор валют

Система должна поддерживать отображение основных мировых валют и предоставлять пользователю возможность самостоятельно
выбирать интересующие валюты.

| Валюта          | Код ISO | Приоритет   | Обоснование               |
|-----------------|---------|-------------|---------------------------|
| Доллар США      | USD     | Критический | Основная резервная валюта |
| Евро            | EUR     | Критический | Валюта Европейского союза |
| Фунт стерлингов | GBP     | Высокий     | Важная торговая валюта    |

## 7. Нефункциональные требования

### 7.1 Требования к производительности

| Характеристика             | Требование                                                      | Метрика               |
|----------------------------|-----------------------------------------------------------------|-----------------------|
| **Актуальность данных**    | Данные должны обновляться не реже чем каждые 15 минут           | ≤ 15 мин              |
| **Скорость отклика**       | Время отображения данных не должно превышать 3 секунд           | ≤ 3 сек               |
| **Пропускная способность** | Система должна поддерживать до 1000 одновременных пользователей | 1000 concurrent users |

### 7.2 Требования к надёжности

| Характеристика            | Требование                                              |
|---------------------------|---------------------------------------------------------|
| **Доступность системы**   | 99.5% времени (не более 36 часов простоя в год)         |
| **Надёжность источников** | Использование минимум 2 независимых источников данных   |
| **Обработка ошибок**      | Graceful degradation при недоступности внешних сервисов |

## 8. Сценарии использования (Use Cases)

### 8.1 UC-001: Просмотр текущего курса валют

**Актор:** Пользователь  
**Предусловие:** Пользователь имеет доступ к приложению  
**Основной сценарий:**

1. Пользователь запускает приложение
2. Пользователь выбирает интересующие валюты
3. Система получает актуальные данные из внешнего API
4. Система отображает текущие курсы выбранных валют
5. Пользователь просматривает информацию

**Постусловие:** Пользователь получил актуальную информацию о курсах валют

### 8.2 UC-002: Анализ динамики курса

**Актор:** Финансовый аналитик  
**Предусловие:** Пользователь авторизован в системе  
**Основной сценарий:**

1. Пользователь выбирает валютную пару для анализа
2. Пользователь указывает временной период
3. Система получает исторические данные
4. Система строит график изменения курса
5. Пользователь анализирует тренды и динамику

**Постусловие:** Пользователь получил визуализацию динамики курса

### 8.3 UC-003: Конвертация валют

**Актор:** Частное лицо  
**Предусловие:** Пользователь знает сумму для конвертации  
**Основной сценарий:**

1. Пользователь выбирает исходную валюту
2. Пользователь выбирает целевую валюту
3. Пользователь вводит сумму для конвертации
4. Система рассчитывает результат на основе актуального курса
5. Система отображает результат конвертации

**Постусловие:** Пользователь получил точную сумму в целевой валюте
