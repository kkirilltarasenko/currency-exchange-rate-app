# Use Case: View Current Exchange Rates

Краткое описание:
Вариант использования позволяет пользователю просматривать текущие курсы валют относительно выбранной базовой валюты.

Предусловия:
- Приложение запущено.
- Пользователь имеет доступ к интерфейсу системы.
- Есть подключение к интернету.

Основной поток:

1. Вариант использования начинается, когда пользователь открывает главное окно приложения.
2. Система отображает выпадающий список "Base Currency" и кнопку "Get Rates".
3. Пользователь выбирает базовую валюту из списка.
4. Пользователь нажимает кнопку "Get Rates".
5. Система отправляет запрос к External Exchange Rate API.
6. Система получает данные о курсах валют.
7. Система отображает таблицу курсов (код валюты, значение курса, дата обновления).
8. Вариант использования завершается.

Альтернативный поток A1 – Ошибка соединения:

5a. API не отвечает или возвращает ошибку.
6a. Система отображает сообщение: "Connection error. Please try again."
7a. Пользователь может нажать кнопку "Retry".
8a. При нажатии "Retry" выполнение возвращается к шагу 5.

Постусловия:
- Курсы валют отображены на экране.
  или
- Пользователь уведомлён об ошибке соединения.

# Use Case: Convert Currency

Краткое описание:
Вариант использования позволяет пользователю конвертировать денежную сумму из одной валюты в другую.

Предусловия:
- Приложение запущено.
- Доступен сервис получения курсов валют.

Основной поток:

1. Вариант использования начинается, когда пользователь открывает раздел "Currency Converter".
2. Система отображает:
    - поле ввода "Amount",
    - список "From Currency",
    - список "To Currency",
    - кнопку "Convert".
3. Пользователь вводит сумму в поле "Amount".
4. Пользователь выбирает исходную валюту.
5. Пользователь выбирает целевую валюту.
6. Пользователь нажимает кнопку "Convert".
7. Система проверяет корректность введённой суммы.
8. Система отправляет запрос к API для получения актуального курса.
9. Система вычисляет результат конвертации.
10. Система отображает результат на экране.
11. Вариант использования завершается.

Альтернативный поток A1 – Некорректный ввод:

7a. Пользователь ввёл пустое значение, текст или отрицательное число.
7b. Система отображает сообщение: "Invalid amount. Please enter a valid number."
7c. Выполнение возвращается к шагу 3.

Альтернативный поток A2 – Ошибка получения данных:

8a. API недоступен.
8b. Система отображает сообщение: "Unable to retrieve exchange rate."
8c. Вариант использования завершается.

Постусловия:
- На экране отображён результат конвертации.
  или
- Пользователь уведомлён об ошибке.

# Use Case: View Historical Data

Краткое описание:
Вариант использования позволяет пользователю просматривать архивные данные курса валют за выбранный период.

Предусловия:
- Приложение запущено.
- Доступен внешний сервис получения данных.

Основной поток:

1. Вариант использования начинается, когда пользователь открывает раздел "Historical Data".
2. Система отображает:
    - список выбора валютной пары,
    - поле выбора даты начала,
    - поле выбора даты окончания,
    - кнопку "Show History".
3. Пользователь выбирает валютную пару.
4. Пользователь указывает диапазон дат.
5. Пользователь нажимает кнопку "Show History".
6. Система проверяет корректность диапазона дат.
7. Система отправляет запрос к API.
8. Система получает исторические данные.
9. Система отображает таблицу архивных значений.
10. Вариант использования завершается.

Альтернативный поток A1 – Неверный диапазон дат:

6a. Дата окончания меньше даты начала.
6b. Система отображает сообщение: "Invalid date range."
6c. Выполнение возвращается к шагу 4.

Альтернативный поток A2 – Отсутствие данных:

8a. API возвращает пустой результат.
8b. Система отображает сообщение: "No data available for selected period."
8c. Вариант использования завершается.

Постусловия:
- Исторические данные отображены.
  или
- Пользователь уведомлён об ошибке.

# Use Case: View Exchange Rate Chart

Краткое описание:
Вариант использования позволяет пользователю просматривать график изменения курса валют за выбранный период.

Предусловия:
- Выбрана валютная пара.
- Доступны исторические данные.

Основной поток:

1. Вариант использования начинается, когда пользователь нажимает кнопку "Show Chart".
2. Система запрашивает исторические данные.
3. Система получает данные от API.
4. Система строит график изменения курса.
5. Система отображает график на экране.
6. Вариант использования завершается.

Альтернативный поток A1 – Ошибка получения данных:

3a. API возвращает ошибку.
3b. Система отображает сообщение: "Unable to build chart."
3c. Вариант использования завершается.

Постусловия:
- График отображён на экране.
  или
- Пользователь уведомлён об ошибке.
